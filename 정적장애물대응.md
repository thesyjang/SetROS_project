정적 장애물 
맵 상에 있던 기존 장애물 / 맵에 없던 장애물 추가 나눠 진행 

## 목표

로봇의 주행 경로 중간에 박스, 가벽, 의자와 같은 정적 장애물을 배치했을 때, Nav2 local costmap이 이를 반영하고 로봇이 장애물을 우회하는지 확인한다.

## 핵심 개념

정적 장애물 회피는 별도의 safety node가 아니라 **Nav2 local costmap**이 담당한다.

구조:

```
LiDAR /scan
    ↓
local costmap
    ↓
Nav2 controller
    ↓
장애물 우회 경로 선택
```

즉 정적 장애물은 다음 방식으로 처리된다.
정적 장애물 감지
→ local costmap에 obstacle 표시
→ controller_server가 충돌 없는 local trajectory 선택
→ 로봇이 장애물을 우회

## 테스트 방법

1. RViz에서 2D Pose Estimate를 설정한다.
2. 로봇 전방 경로 중간에 박스를 배치한다.
3. 박스 뒤쪽 또는 옆쪽에 Nav2 Goal을 찍는다.
4. 로봇이 박스를 피해서 우회하는지 확인한다.

권장 장애물:

- 박스
- 하드보드
- 넓은 판
- LiDAR 높이에 걸리는 물체

피해야 할 장애물:

- 너무 얇은 의자 다리
- 유리
- 검은색 흡광 물체
- LiDAR 높이보다 낮은 물체

---

##  확인 항목

| 확인 항목 | 정상 기준 |
| --- | --- |
| local costmap | 장애물 주변이 costmap에 표시됨 |
| global path | 목표 지점까지 경로가 생성됨 |
| local trajectory | 장애물을 피해 꺾임 |
| 실제 주행 | 로봇이 장애물을 우회 |
| 충돌 여부 | 장애물과 충돌하지 않음 |

---

##  정적 장애물 회피 결과

테스트 결과, 로봇은 정적 장애물을 정상적으로 감지하고 우회하였다.

정리:
```
정적 장애물 회피 성공
- LiDAR scan 정상 반영
- local costmap 장애물 표시 정상
- Nav2 경로 생성 정상
- 로봇 우회 주행 정상
```

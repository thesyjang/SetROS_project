2. 장애물 회피가 잘 드러나는 공간
    1. SLAM은 정적환경에서 하고, 장애물 회피 테스트는 Nav2단계에서
3. 통로 폭이 적당한 곳
    1. 너무 넓으면 장애물을 피할 필요성 x
    2. 너무 좁으면 경로를 못찾거나 costmap상 정지
4. 코너가 있는 공간
5. SLAM용 고정 구조물
    1. localization을 위해서 SLAM 지도 자체가 좋아야 함 → 고정 구조물이 많으면 좋음
    2. 벽, 문틀, 기둥, 코너
    3. 유리벽, 거울은 피하는 게 좋음
6. 정적 장애물과 일시적 장애물 (맵에 없는) 분리해서 테스트
    1. 테스트 지점을 분리하는게 좋음
    2. 정적 장애물 구간, 일시적 장애물 구간

# 실제 위치 선정

 평평한 바닥이 있는, 2m x 2m 여유있게 사용 가능한 공간 선정 → 바닥 테이핑으로 위치 표시 → 가벽 및 장애물 설치 (박스 등)

1. 유동인구 적은 시간대 넓은 복도, 로비 
2. 강의실 (책상 이슈)
3. 주변 연습실

- 공간 제한 - 실제 공간 사용하되 가벽 설치 고려
    
    터틀봇 3 LiDAR 유효거리가 약 3-4미터
    → 벽과벽 사이 거리가 3m 내로 들어오는 공간
    
    모든 기둥/벽이 똑같고 배치까지 규칙적이면 로봇은 자기 위치 파악 못함 (상대적인 맥락으로 장애물 인식)
    → 불규칙 배치 or 불규칙한 요철 같은 랜드마크 만들어줘야함 
    실제 공간 (강의실, 복도) 통으로 사용하기엔 공간 크기 제한에 어려움이 있음
    
    인플레이션 설정 잘 맞물리면 벽이 뚫려 있어도 벽처럼 사용 가능함. 예를 들어 터틀봇이 ‘20cm 이하로는 못지나가’ 라고 판단하면, 가벽을 듬성듬성 쳐도 공간 제한 가능.
